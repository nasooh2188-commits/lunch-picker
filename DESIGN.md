# 🍽️ 점심 뭐 먹지? - 설계서

## 1. 프로젝트 개요
- **목적:** 매일 점심 메뉴 고민 해결 — 키워드 추천 + 결정 게임
- **플랫폼:** 모바일 웹 (스마트폰 최적화)
- **배포:** GitHub Pages (무료, 정적 사이트에 최적, 별도 빌드 불필요)
- **기술스택:** HTML + CSS + Vanilla JS (프레임워크 없이 가볍게)

---

## 2. 주요 기능

### 2-1. 키워드 기반 식당 추천
- 기분/상황 키워드를 선택하면 해당 키워드에 맞는 식당 추천
- **키워드 카테고리:**
  - **음식종류:** 한식, 중식, 일식, 양식, 분식, 동남아
  - **조리방식:** 면요리, 구이, 튀김, 국물요리
  - **기분/상황:** 매운, 든든한, 가벼운, 시원한, 뜨거운, 보양식, 저렴한

### 2-2. 결정 게임 (4종)
1. **룰렛** — 돌려서 멈추는 곳이 오늘의 식당
2. **사다리타기** — 사다리 선택해서 결과 확인
3. **랜덤뽑기** — 카드 뒤집기 방식으로 랜덤 선택
4. **핀볼** — 공을 떨어뜨리면 핀에 부딪히며 식당 칸에 도착 (파칭코 스타일, Canvas)

## 5. 결정 게임 상세

### 5-1. 룰렛
- 원형 룰렛에 식당명 배치 (전체 또는 키워드 필터된 목록)
- 터치/클릭으로 회전 → 감속 → 멈춘 위치의 식당 선택
- 회전 애니메이션 + 효과음(옵션)

### 5-2. 사다리타기
- 4~6개 세로줄 + 랜덤 가로줄 생성
- 출발점 선택 → 애니메이션으로 경로 따라감 → 도착 식당 공개

### 5-3. 랜덤뽑기
- 카드 뒷면이 보이는 상태로 배열
- 하나 터치하면 뒤집히며 식당 공개
- 나머지 카드도 순차적으로 공개

### 5-4. 핀볼 (파칭코 스타일)
- Canvas 기반 물리 시뮬레이션
- 상단에서 공을 떨어뜨리면 중간의 핀(못)에 부딪히며 좌우로 튕김
- 하단에 식당 칸(슬롯)이 배치되어 공이 최종 도착한 칸의 식당 선택
- 참여 식당 수에 따라 슬롯 수 자동 조절 (5~8개)
- 공 떨어뜨리는 위치를 터치로 선택 가능
- 물리 엔진: 중력 + 핀 충돌 반사 (간단한 2D 물리)

## 6. 추가 기능
- **네이버지도 연결:** 식당 선택 시 네이버지도 앱/웹으로 바로 이동
- **결과 공유:** 선택된 식당을 카카오톡 등으로 공유 (선택)
- **최근 선택 기록:** 최근 5일 선택 기록 표시 (localStorage)
- **식당 필터:** 게임에 참여할 식당을 체크박스로 필터링

## 7. 디자인 방향
- 밝고 재밌는 느낌 (음식 이모지 활용)
- 큰 버튼, 큰 글씨 (모바일 터치 최적화)
- 게임 화면은 애니메이션으로 재미 요소
- 색상: 따뜻한 오렌지/노란색 계열 (식욕 자극)
